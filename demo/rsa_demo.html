4攻击者视角:为什么破不了

窃听者手里有公钥 (n, e) 和密文 c。想解密就要私钥 d;算 d 需要 φ(n)=(p−1)(q−1);而算 φ 就必须把 n 分解回 p、q。

✅ 你(有 p、q):秒算 d → 解密
❌ 攻击者(只有 n):必须先分解 n → 大数分解难题,做不到

这里 n 小,攻击者也能分解出来 —— 说明「n 太小 = 不安全」。真实 RSA 的 n 有 2048~4096 位,这步就变成不可能。和 ECC 的离散对数一样,都是「正向易、反向难」的陷门。
